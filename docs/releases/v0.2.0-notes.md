## Verified (no-root gates)

All of the following were green on the release branch before tag:

```bash
make vet && make test && make smoke && make replay && make helm-lint
```

These cover:

- SQLite single-replica ingest, fingerprint merge, diff, API, dashboard embed
- Transactional `MergeFingerprint` / `UpsertAlert` concurrency tests
- Replay corpus (real npm supply-chain attack scenarios)
- Helm HA template guards (`collector.replicas > 1` requires `postgres.dsn`)
- Enforcement pipeline smoke (verdict compile, runtime switch) without a kernel

Optional when Docker is available:

```bash
make ha-smoke   # two collectors + Postgres; fingerprint/alert dedup; skips if no docker
```

`WithLeader` (digest/retention singleton loops) is covered by `internal/store`
unit tests; per-replica digest tickers are out of phase in a two-process smoke,
so digest doubling is verified in staging Helm, not this script.

## Requires a human (not automated in CI)

| Gate | Command / action | Why |
|---|---|---|
| Live eBPF + LSM | `sudo make e2e` | Needs root/CAP_BPF and `CONFIG_BPF_LSM` + `lsm=bpf` for block-mode proof |
| Two-replica HA | `make ha-smoke` or staging Helm with `collector.replicas=2` + Postgres | SQLite is single-writer — **two collectors against one SQLite file is invalid** |
| Tag + images | `git tag v0.2.0` + push per `docs/development.md` | Intentionally manual |
| Helm upgrade | `helm upgrade …` with production values | Operator-owned |

Record the environment and outcome for HA and `sudo make e2e` in this file when
executed before the public tag.

## Upgrade notes

- **Default posture unchanged:** detection-only; `enforce.enabled=false`; auth on in Helm.
- **HA:** set `postgres.dsn` before raising `collector.replicas` above 1.
- **Warn before block:** run `action: warn` rules during pilot; review would-block
  counts vs alert budget before enabling `block` + enforcement labels.
- **Dashboard:** if upgrading from a build that predates Phase 3–6 UI, no extra
  step — `internal/api/ui/dist/` is embedded in the collector binary.

## Known limitations

- SSE is per-replica in HA mode; REST remains authoritative (`docs/deployment.md`).
- Redis was not added; Postgres transactional upserts are the HA write path.
- Tier-2 (flagless V8) attribution remains parked until evidence changes the GO/NO-GO.
